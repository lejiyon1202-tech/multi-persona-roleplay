## 0. character-select.html 수정 사항 (Phase A 권고 반영)

### 0-1. inline onclick → addEventListener 전환 (필수 — helmet CSP 충돌 방지)
**현재 (Phase A):**
```html
<button class="cta-btn" onclick="startChat(event, '이모델 파트장', 'role-lead', '03')">
```
```js
function goToChat() { ... }
```

**Phase B 수정안:**
```html
<button class="cta-btn" data-char-name="이모델 파트장" data-role-class="role-lead" data-number="03">
  대화 시작 <svg>...</svg>
</button>
```
```js
// 모든 이벤트 addEventListener로 분리
document.querySelectorAll('.cta-btn').forEach(btn => {
  btn.addEventListener('click', e => {
    e.stopPropagation();
    const { charName, roleClass, number } = btn.dataset;
    startChat(charName, roleClass, number);
  });
});
document.getElementById('startBtn').addEventListener('click', goToChat);
```

### 0-2. opacity 0.55 표준 통일
```css
/* 합의서 v1 명시값 일치 */
.char-card.context-only { opacity: 0.55; }
```

### 0-3. hint-dot.context 대비 보강
```css
/* AAA 기준 미달 → AA 달성을 위해 색상 강화 */
.hint-dot.context { background: #6b7280; }  /* #9ca3af → #6b7280 */
```
- 라이트 테마 #f0f2f8 기준: #6b7280 대비 약 4.7:1 (WCAG AA 통과)

## 1. index.html — 시나리오 선택 화면

### 목적
학습자가 롤플레잉 시나리오를 선택하는 진입 화면.

### UI 구성

```
┌─────────────────────────────────────────────────────────────────┐
│  헤더: "어떤 상황을 연습하시겠습니까?"                              │
│  설명: 상황을 선택하면 대화할 인물을 고를 수 있습니다.              │
│                                                                 │
│  ┌──────────────────┐  ┌──────────────────┐                     │
│  │ [케이스 배지]     │  │ [케이스 배지]     │                     │
│  │                  │  │                  │                     │
│  │  Case 01         │  │  Case 02         │                     │
│  │  조직 방향성 정렬 │  │  성과 압박 상황  │                     │
│  │                  │  │                  │                     │
│  │  등장 인물: 6명   │  │  등장 인물: 6명  │                     │
│  │  [시작하기]      │  │  [시작하기]      │                     │
│  └──────────────────┘  └──────────────────┘                     │
└─────────────────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트

| 요소 | 사양 |
|------|------|
| 시나리오 카드 | 케이스 번호 배지 + 제목(20px+) + 설명(15px) + 등장인물 수 + "시작하기" 버튼 |
| hover | translateY(-4px) + 그림자 + 테두리 강화 |
| 선택 CTA | "시작하기" 클릭 → character-select.html?scenario_id=X |
| 시나리오 없을 때 | 빈 상태 컴포넌트 (관리자 문의 안내) |
| 케이스 배지 | "CASE 01", "CASE 02" — 역할 뱃지 디자인 차용 |

### API 연동
```js
// GET /api/scenarios
fetch('/api/scenarios')
  .then(r => r.json())
  .then(data => renderScenarioCards(data.scenarios));
```

### 파일: `public/index.html`
### CSS: `public/css/scenario-select.css` (신규) 또는 design-system.css 확장

## 2. chat.html — 1:1 롤플레잉 채팅 화면

### 목적
선택된 캐릭터와 학습자가 1:1 대화하는 핵심 화면.

### UI 구성

```
┌──────────────────────────────────────────────────────────────────┐
│  [뒤로가기]  |  캐릭터 카드 (미니: 역할뱃지 + 이름 + 상황 요약) │  [3테마]  │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [AI 메시지 버블] (왼쪽 정렬, 배경색=bg-quote)                  │
│                                                                  │
│                      [학습자 메시지 버블] (오른쪽, 배경=역할색)  │
│                                                                  │
│  [AI 메시지 버블]                                                │
│                                                                  │
│  ── 감정 단계 전환 인디케이터 (선택적 표시) ───────────────────  │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│  [입력창                                      ] [전송 ▶]        │
│  [대화 종료 + 평가 받기]                                         │
└──────────────────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트

| 요소 | 사양 |
|------|------|
| 헤더 미니 카드 | 역할 뱃지 + 이름 + 핵심 마인드 1줄 (접힘/펼침 토글) |
| AI 버블 | 왼쪽 정렬 + `--bg-quote` 배경 + 역할 색상 좌측 세로선 |
| 학습자 버블 | 오른쪽 정렬 + `--role-*` 연한 배경 |
| 입력창 | border-radius 12px + Enter 전송 + Shift+Enter 줄바꿈 |
| 전송 버튼 | 역할 색상 배경 + 로딩 스피너 (AI 응답 대기 중) |
| 대화 종료 | 턴 수 표시 + "평가 받기" 버튼 (최소 3턴 이후 활성화) |
| 감정 단계 인디케이터 | emotion_stages 현재 단계 작은 뱃지 (선택적 노출) |
| 타이핑 인디케이터 | AI 응답 대기 중 ". . ." 애니메이션 |

### 접근성
- `aria-live="polite"` on 메시지 영역
- 입력창 `role="textbox"` + `aria-label`
- 전송 버튼 `aria-label="메시지 전송"`

### API 연동
```js
// POST /api/chat
// body: { session_id, message }
// 스트리밍 응답 (EventSource 또는 ReadableStream)
```

### 파일: `public/chat.html`
### CSS: `public/css/chat.css` (신규)

## 3. report.html — 평가 리포트 화면

### 목적
세션 종료 후 학습자 발화 평가 + 다중 세션 비교 리포트.

### UI 구성

```
┌──────────────────────────────────────────────────────────────────┐
│  탭: [이번 세션 리포트]  [캐릭터별 비교]                          │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ── 단일 세션 탭 ──────────────────────────────────────────────  │
│                                                                  │
│  [총점 카드] 18.5 / 25점    [등급] "아쉽지만..."                 │
│                                                                  │
│  R-26 (15점) ──────────────────── [점수 바]                     │
│  R-27 (5점)  ──────────────────── [점수 바]                     │
│  R-28 (5점)  ──────────────────── [점수 바]                     │
│                                                                  │
│  [항목별 세부 피드백 카드 리스트]                                 │
│                                                                  │
│  ── 캐릭터별 비교 탭 ───────────────────────────────────────── │
│                                                                  │
│  [레이더 차트: 캐릭터 A vs B vs C]                               │
│  [캐릭터별 강약점 비교 테이블]                                    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트

| 요소 | 사양 |
|------|------|
| 총점 카드 | 점수 크게(36px+) + 등급 뱃지 (됐어!/아쉽지만.../느낌이 안 와) |
| 점수 바 | CSS 애니메이션 width 확장 (0 → 최종점수) |
| 등급 색상 | ≥95% 녹색 / 80~94% 앰버 / <80% 빨강 |
| 세부 피드백 | R-26/R-27/R-28 축별 아코디언 카드 |
| 캐릭터 비교 | SVG 레이더 차트 (최대 3개 세션 동시 표시) |
| "다시 도전" CTA | character-select.html로 이동 + 선택 캐릭터 미선택 상태로 |

### API 연동
```js
// GET /api/sessions/:id/report         단일 세션
// GET /api/sessions/compare?learner_id&scenario_id   다중 비교
```

### 파일: `public/report.html`
### CSS: `public/css/report.css` (신규)

## 4. admin.html — 관리자 대시보드

### 목적
시나리오·캐릭터 풀 CRUD 관리.

### UI 구성

```
┌──────────────────────────────────────────────────────────────────┐
│  사이드바: [시나리오 관리]  [학습자 현황]  [세션 통계]             │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ── 시나리오 관리 ────────────────────────────────────────────  │
│  [+ 새 시나리오 추가] 버튼                                       │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ Case 01 조직 방향성 정렬              [수정] [캐릭터] [삭제] │   │
│  │ 캐릭터 6명 · 세션 42개                                    │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  ── 캐릭터 관리 (시나리오 선택 시) ────────────────────────────  │
│  [카드 그리드: character-select.html 디자인 동일]                │
│  각 카드에 [수정] [삭제] 버튼 오버레이                           │
│  [+ 새 캐릭터 추가] 버튼                                         │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트

| 요소 | 사양 |
|------|------|
| 시나리오 목록 | 테이블형 카드 + 수정/캐릭터 관리/삭제 3버튼 |
| 캐릭터 카드 | character-select.html과 동일 디자인 + 수정/삭제 오버레이 |
| 모달 에디터 | 시나리오·캐릭터 신규/수정 → 슬라이드인 모달 |
| 폼 필드 | 역할 뱃지 드롭다운(ENUM 4종) + is_selectable 토글 + persona_prompt 텍스트에어리어(큼직하게) |
| 삭제 확인 | dialog confirm 사용 (onclick 아닌 addEventListener) |
| 학습자 현황 | 세션 수·총점 평균 테이블 |

### API 연동
```js
// GET/POST /api/admin/scenarios
// GET/PUT/DELETE /api/admin/scenarios/:id
// GET/POST /api/admin/scenarios/:id/characters
// GET/PUT/DELETE /api/admin/scenarios/:id/characters/:cid
```

### 파일: `public/admin.html`
### CSS: `public/css/admin.css` (신규)
